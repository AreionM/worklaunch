# WorkLaunch

A Windows system tray app that launches all your work tools — apps and websites — in one click. Lives quietly in the taskbar, never opens a duplicate if something's already running.

## What it does

- Sits in the system tray (bottom-right taskbar area)
- **Left-click** the tray icon → opens Settings
- **Double-click** the tray icon → launches all enabled items immediately
- **Right-click** the tray icon → context menu with Launch All / Settings / Quit
- Before launching each app, checks if the process is already running via `tasklist` and skips it if so
- Shows a Windows toast notification reporting what was launched vs skipped
- Optional: start automatically with Windows

## Default configured items

| App | Process checked |
|-----|----------------|
| Opera GX | `opera.exe` |
| Discord | `Discord.exe` |
| Microsoft Teams | `ms-teams.exe` |
| Power BI Desktop | `PBIDesktop.exe` |
| Claude Desktop | `claude.exe` |

Paths are auto-detected for common install locations. If an app isn't found at the default path, edit it in Settings.

## How to install (pre-built)

1. Download the latest `WorkLaunch-Setup-x.x.x.exe` from [Releases](../../releases)
2. Run the installer — choose per-user install (default)
3. WorkLaunch starts and appears in your system tray
4. Right-click the tray icon → **Settings** to configure your apps

## How to build from source

### Prerequisites

- [Node.js](https://nodejs.org/) v18 or newer
- [Git](https://git-scm.com/)

### Steps

```bash
# Clone the repo
git clone https://github.com/AreionM/worklaunch.git
cd worklaunch

# Install dependencies
npm install

# Generate the app icon
node scripts/generate-icon.js

# Run in development mode (launches the Electron app live)
npm start

# Build production bundles only (no installer)
npm run build

# Build + create Windows installer (.exe)
npm run dist:win
```

The installer will be in `dist/`.

## How to add a new launch item

### Via the Settings UI

1. Click the tray icon (or right-click → Settings)
2. Click **+ Add Item**
3. Fill in:
   - **Name**: display label
   - **Type**: `app` or `website`
   - **Path**: full path to the `.exe`, or a URL for websites
   - **Process Name**: the `.exe` filename as it appears in Task Manager (apps only)
   - **Arguments**: optional command-line flags
4. Click **Add Item**, then **Save Config**

### Via config file directly

Config lives at `%APPDATA%\worklaunch\config.json`. Edit it while the app is closed:

```json
{
  "items": [
    {
      "id": "slack",
      "name": "Slack",
      "type": "app",
      "path": "C:\\Users\\YourName\\AppData\\Local\\slack\\slack.exe",
      "args": [],
      "processName": "slack.exe",
      "enabled": true
    },
    {
      "id": "my-dashboard",
      "name": "Dashboard",
      "type": "website",
      "path": "https://dashboard.example.com",
      "processName": "",
      "enabled": true
    }
  ],
  "autoStart": false
}
```
